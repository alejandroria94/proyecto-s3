Módulo cursos-service

Página explicativa del servicio de cursos y del valor de repetir el patrón arquitectónico con una nueva entidad.

Propósito del módulo

El servicio de cursos cumple una función estratégica en el proyecto: demostrar que el equipo no depende de copiar un CRUD ya existente, sino que puede construir uno nuevo siguiendo el mismo patrón arquitectónico. Este servicio es la prueba de que el estudiante comprendió el molde técnico y puede reproducirlo con otra entidad.

Responsabilidades

  • Registrar cursos con un código único.
  • Listar y consultar la oferta académica.
  • Actualizar los datos de un curso.
  • Eliminar cursos según permisos.

Descripción de la entidad

La entidad Curso representa una unidad académica ofertada por la institución. Además del nombre y la descripción, conviene incluir un código que identifique el curso de forma única y un docente responsable para dar contexto a la información.

@Entity
public class Curso {
    private Long id;
    private String codigo;
    private String nombre;
    private String descripcion;
    private Integer creditos;
    private String docenteResponsable;
}

Aprendizajes que refuerza

  • Diseño de DTO para entrada y salida.
  • Validaciones de unicidad y obligatoriedad.
  • Repetición disciplinada del patrón por capas.
  • Uso de las mismas convenciones del resto del sistema.

Puntos que deben quedar bien explicados en clase

  • Por qué el código del curso debe ser único.
  • Por qué conviene mantener el DTO separado de la entidad.
  • Cómo se reutiliza el mismo mecanismo de seguridad ya aplicado en estudiantes-service.
