Proyecto Final Guiado del Curso

Portal principal con la navegación completa del proyecto final y las páginas explicativas por módulo.

Bienvenida

Este sitio reúne la guía completa del proyecto final del curso. Aquí no solo se describe qué debe construir el estudiante, sino también por qué se construye así, cuál es la secuencia recomendada de desarrollo y cómo se conecta cada módulo con los aprendizajes del laboratorio base.

Idea central: el proyecto final no parte de cero. Es una continuación natural del Lab 04, ampliando la autenticación, la seguridad, la comunicación entre servicios y la orquestación con Docker.

Rutas disponibles

Guía general del proyecto

Contiene el caso, objetivos, competencias, fases, arquitectura, entregables, rúbrica, pruebas y checklist final.

Módulo auth-service

Explica la autenticación, el uso de BCrypt, la generación de JWT, la validación del token y la carga inicial de usuarios.

Módulo estudiantes-service

Describe cómo transformar el CRUD base del laboratorio en un microservicio protegido que reutiliza la arquitectura por capas.

Módulo cursos-service

Presenta la construcción de un nuevo servicio siguiendo el mismo patrón técnico del curso para fortalecer la repetición arquitectónica.

Módulo matriculas-service

Explica las validaciones de negocio, el consumo con OpenFeign y el control de duplicidad y consistencia antes de registrar una matrícula.

Docker, Swagger y pruebas

Detalla cómo ejecutar todo el sistema, qué validar en Swagger y qué evidencias presentar en la entrega final.

Qué encontrará el estudiante en este sitio

  • Una descripción pedagógica del propósito del proyecto y del valor de cada módulo.
  • Una secuencia guiada para no perderse durante el desarrollo.
  • Bloques de código base para arrancar cada servicio.
  • La lógica de roles esperada y las pruebas mínimas obligatorias.
  • Orientaciones para documentar el trabajo y empacarlo con Docker.
