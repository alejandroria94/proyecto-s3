Módulo auth-service

Guía de trabajo para construir el servicio de autenticación del proyecto.

Propósito del módulo

El objetivo de auth-service es separar la autenticación del resto del sistema. En lugar de repetir verificación de usuario y contraseña en todos los microservicios, este servicio centraliza esa responsabilidad y entrega un token reutilizable por los demás módulos.

Meta de aprendizaje: comprender la diferencia entre autenticación, autorización, cifrado de contraseña y validación de token.

Qué debe lograr el estudiante

  • Recibir credenciales en /auth/login.
  • Buscar el usuario en base de datos.
  • Comparar la contraseña con BCrypt.
  • Generar un JWT con usuario y rol.
  • Exponer /auth/validate para validar el token.

Prerrequisitos

  • Tener Java 21 y Maven funcionando.
  • Comprender la estructura por capas del curso.
  • Conocer DTO, endpoints REST y validación básica.

Pasos guiados

1. Cree el proyecto con Web, JPA, Validation, Security, H2 y Swagger.
2. Defina la entidad Usuario con username, password y role.
3. Cree el repositorio y el método para buscar por usuario.
4. Implemente el DTO de entrada y el DTO de respuesta del login.
5. Configure BCrypt para almacenar contraseñas cifradas.
6. Implemente la utilidad JWT.
7. Cree los endpoints /auth/login y /auth/validate.
8. Agregue usuarios de prueba y valide el flujo completo en Swagger.

Endpoints mínimos

EndpointMétodoDescripción
/auth/loginPOSTRecibe usuario y contraseña y retorna un JWT.
/auth/validateGETValida el token enviado por encabezado.

Buenas prácticas que debe aplicar

  • No guardar contraseñas en texto plano.
  • No devolver la contraseña en la respuesta.
  • Permitir acceso libre solo a Swagger, H2 Console y los endpoints de autenticación.
  • Incluir el rol dentro del token para que otros servicios puedan autorizar acciones.

Evidencias esperadas

  1. Login exitoso con usuario válido.
  2. Login fallido con credenciales incorrectas.
  3. Validación correcta de un token.
  4. Código o evidencia de los usuarios de prueba cargados.

Errores comunes

  • Olvidar cifrar la contraseña.
  • Configurar mal el secreto JWT.
  • No incluir el rol dentro del token.
  • Bloquear por error el acceso al endpoint de login.

Checklist del módulo

  • Existe entidad Usuario correctamente configurada.
  • El login genera JWT.
  • El endpoint de validación responde correctamente.
  • Swagger documenta el módulo.
  • Hay al menos un caso positivo y uno negativo probados.
