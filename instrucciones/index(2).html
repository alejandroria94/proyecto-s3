Proyecto Final Guiado del Curso

Portal principal con la navegación completa del proyecto final y las páginas de apoyo por módulo.

Bienvenida

Este portal organiza el proyecto final del curso como una ruta de trabajo clara para el estudiante. La idea no es solo mostrar qué archivos debe construir, sino orientar el proceso completo: qué hacer primero, qué validar, qué evidencias obtener y cómo conectar todos los módulos hasta llegar a una solución funcional.

Punto de partida: antes de avanzar, descargue o clone el proyecto base desde el repositorio del curso y revise el laboratorio sobre el que se apoya el proyecto final.

Abrir repositorio del proyecto base

Ruta sugerida de trabajo

1. Lea primero la guía general para entender el caso, el alcance y los entregables.
2. Implemente auth-service y verifique que el login genere un token válido.
3. Construya estudiantes-service reutilizando el CRUD del laboratorio y protéjalo con JWT.
4. Cree cursos-service repitiendo la arquitectura por capas con una nueva entidad.
5. Desarrolle matriculas-service, que integra reglas de negocio y comunicación entre servicios.
6. Documente, dockerice y pruebe el sistema completo con la guía final de Docker, Swagger y pruebas.

Páginas del sitio

Guía general del proyecto

Explica el caso, el alcance, la secuencia de desarrollo, los roles, la rúbrica y el checklist final.

Módulo auth-service

Guía para construir el servicio de autenticación, generar JWT y validar el token.

Módulo estudiantes-service

Guía para convertir el CRUD base en un microservicio protegido y bien organizado.

Módulo cursos-service

Guía para replicar el patrón arquitectónico con una nueva entidad y nuevas reglas.

Módulo matriculas-service

Guía para implementar el servicio integrador con OpenFeign y validaciones cruzadas.

Docker, Swagger y pruebas

Guía de empaquetado, orquestación, pruebas mínimas y evidencias de entrega.

Qué debe entregar el estudiante al final

  • Repositorio con los cuatro microservicios.
  • Endpoints funcionales y documentados en Swagger.
  • Autenticación con JWT y control de roles.
  • Dockerfile por servicio y docker-compose.yml.
  • Evidencias de pruebas funcionales.
  • README o explicación mínima de ejecución.
